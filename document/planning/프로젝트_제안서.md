# 1. 프로젝트 개요

## 1.1 프로젝트 명

- webOS의 첫 화면을 디자인 하라

## 1.2 프로젝트 목표

- 본 프로젝트는 webOS 환경에서 사용자 인식과 맞춤형 상호작용을 지원하는 차세대 홈 화면을 구현하는 것을 목표로 한다
- 얼굴 인식 기능을 통해 로그인 과정의 번거로움을 제거하고, 사용자별 개인화된 콘텐츠 및 앱 환경을 자동으로 구성하여 모든 세대가 직관적으로 사용할 수 있는 접근성을 확보한다.
- 또한 가정 구성원 간의 소통을 위한 메모 및 메시지 기능을 추가하여, TV를 단순한 콘텐츠 소비 기기를 넘어 가정 내 커뮤니케이션 허브로 발전시키고 IoT 생태계와의 연결 기반을 마련한다.

## 1.3 프로젝트 범위

본 프로젝트는 아래와 같은 기능을 포함한다.

- 필수 기능
  - webOS App 실행 및 App List 관리
  - 미디어 재생 / 볼륨 조절
  - 사용자 계정 로그인
  - 시계 / 날씨 표시
- 선택 / 심화 기능
  - 얼굴 인식 로그인
  - 캐릭터 ‘빈버드’ 인터렉션
  - 메모장 기능
  - 앱 즐겨찾기 기능
- 비기능 요구사항
  - **성능**: CPU 사용률 30% 이하, API 응답 시간 1초 이내
  - **메모리**: Memory Leak 없음
  - **보안**: JWT 인증, Rate Limiting (IP당 20req/s), 파일 업로드 크기 제한 (10MB)
  - **안정성**: Database Connection Pool 관리, 에러 로깅 (Pino)
  - **확장성**: Feature-Sliced Design 적용, OpenAPI 문서화

## 1.4 기술 스택

### 데이터베이스 스키마 (PostgreSQL)

```sql
-- 사용자 테이블
CREATE TABLE users (
  id SERIAL PRIMARY KEY,
  username VARCHAR(50) UNIQUE NOT NULL,
  face_encoding TEXT NOT NULL,  -- 128차원 얼굴 특징 벡터 (JSON)
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
  updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

-- 사용자별 앱 정렬 순서 테이블
CREATE TABLE user_app_orders (
  id SERIAL PRIMARY KEY,
  user_id INTEGER REFERENCES users(id) ON DELETE CASCADE,
  app_order JSONB NOT NULL,  -- webOS 앱 ID 배열 (예: ["com.webos.app.browser", "com.webos.app.settings"])
  updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
  UNIQUE(user_id)
);

-- 메모 테이블
CREATE TABLE memos (
  id SERIAL PRIMARY KEY,
  user_id INTEGER REFERENCES users(id) ON DELETE CASCADE,
  title VARCHAR(255) NOT NULL,
  content TEXT NOT NULL,
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
  updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

-- 즐겨찾기 테이블
CREATE TABLE favorites (
  id SERIAL PRIMARY KEY,
  user_id INTEGER REFERENCES users(id) ON DELETE CASCADE,
  app_id VARCHAR(100) NOT NULL,  -- webOS 앱 ID (예: "com.webos.app.browser")
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
  UNIQUE(user_id, app_id)
);

-- 성능을 위한 인덱스
CREATE INDEX idx_memos_user_id ON memos(user_id);
CREATE INDEX idx_favorites_user_id ON favorites(user_id);
CREATE INDEX idx_user_app_orders_user_id ON user_app_orders(user_id);
```

### 얼굴 인식 기술 상세

**사용 모델:**

- **SSD MobileNetV1**: 얼굴 감지 (~4MB)
- **Face Landmark 68**: 얼굴 랜드마크 감지 (~348KB)
- **Face Recognition Net**: 128차원 얼굴 특징 추출 (~6MB)

**처리 과정:**

1. 프론트엔드에서 webOS Luna Camera API로 얼굴 사진 촬영
2. JPEG 이미지를 Multipart로 백엔드 전송
3. face-api.js로 얼굴 감지 및 128차원 특징 벡터 추출
4. DB에 저장된 모든 사용자의 얼굴 벡터와 유클리드 거리 비교
5. 유사도 0.6 이상이면 동일인으로 판단
6. JWT 토큰 발급 (만료 시간: 24시간)

**예상 성능:**

- 얼굴 감지: ~200ms
- 특징 추출: ~300ms
- DB 비교 (100명 기준): ~50ms
- 총 응답 시간: ~600ms

# 2. 기대효과 및 차별화 포인트

### [사용자 편의성과 접근성 강화]

- 별도의 리모컨 조작이나 비밀번호 입력 없이 얼굴 인식만으로도 자동 로그인이 가능하며 모든 연령대가 쉽게 이용 가능함

### [개인화된 사용자 경험 제공]

- 얼굴 인식을 통해 사용자별로 즐겨찾기 앱, 콘텐츠 추천, 시청 기록 복원이 자동으로 활성화됨.
- 아동 사용자의 경우 유해 콘텐츠 접근 차단을 통해 안전하고 신뢰할 수 있는 환경을 조성함.

### [가정 내 소통 및 지속적 이용 촉진]

- 가족 간 메모·메시지 공유 기능을 통해 TV를 가족 커뮤니케이션 플랫폼으로 확장.
- 사용자 위젯 커스터마이징과 메모 공유로 TV 이용 빈도와 체류 시간 증가를 유도함.

### [기술적·서비스적 확장성]

- 얼굴 인식, 빈버드 인터렉션을 통해 기술적 차별성 사용자 경험을 확보함.
- 향후 IoT 기기나 모바일 기기와의 연동으로 홈케어 및 스마트홈 통합 서비스로 확장 가능한 구조를 구축함.
